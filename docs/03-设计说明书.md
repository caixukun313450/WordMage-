# WordMage 设计说明书

**版本**：1.0  
**日期**：2026-06-10

## 1. 系统架构

```
┌─────────────┐     HTTP      ┌──────────────────┐     HTTPS     ┌─────────────┐
│  浏览器前端   │ ──────────► │  Spring Boot 后端  │ ──────────► │  大模型 API   │
│  (静态页面)  │ ◄────────── │  /api/story       │ ◄────────── │ 智谱/通义    │
└─────────────┘               └──────────────────┘               └─────────────┘
       │
       ▼
┌─────────────┐
│ localStorage │
│  学习记录    │
└─────────────┘
```

## 2. 模块设计

### 2.1 后端模块

| 类 | 职责 |
|----|------|
| `StoryController` | 接收生成请求，返回 JSON |
| `StoryService` | 构建 Prompt、调用 LLM、解析响应、演示模式 |
| `LlmProperties` | 读取 API Key、模型、厂商配置 |
| `WebConfig` | CORS 配置 |

### 2.2 前端模块

| 文件 | 职责 |
|------|------|
| `app.js` | 主流程：输入、生成、UI 刷新 |
| `storage.js` | localStorage 封装：星星、故事、词频 |
| `tts.js` | SpeechSynthesis 封装 |
| `game.js` | 单词配对游戏逻辑 |

## 3. 数据设计

### 3.1 localStorage 键

| 键 | 类型 | 说明 |
|----|------|------|
| `wm_stars` | number | 星星总数 |
| `wm_last_star_date` | string | 上次获星日期 |
| `wm_stories` | array | 故事历史（最多 100 条） |
| `wm_story_count` | number | 故事生成计数 |
| `wm_treasure_unlocked` | boolean | 宝藏是否已解锁 |

### 3.2 故事记录结构

```json
{
  "id": 1718000000000,
  "words": ["apple", "happy"],
  "title": "A Happy Apple Day",
  "date": "2026-06-10T08:00:00.000Z"
}
```

## 4. 界面设计

- **Tab 1 创作故事**：每日词、输入区、词库、故事展示、TTS 按钮
- **Tab 2 复习列表**：7 天单词列表，点击朗读
- **Tab 3 学习报告**：统计数字 + 词频排行

## 5. 关键流程

### 5.1 故事生成时序

```
用户点击生成 → 校验单词 → POST /api/story → 渲染故事
    → 保存历史 → 尝试加星 → 刷新 UI → (每3次) 打开配对游戏
```

### 5.2 LLM Prompt 设计

- 角色：小学 3-6 年级英语教师
- 约束：80-120 词、积极主题、JSON 格式输出
- 字段：title、storyEn、storyZh

## 6. 安全设计

- API Key 仅存服务端环境变量，不暴露给前端
- 输入单词过滤非字母字符
- 故事内容通过 Prompt 约束适龄

## 7. 部署

- 单 JAR 部署：`java -jar WordMage-0.0.1-SNAPSHOT.jar`
- 静态资源内嵌于 Spring Boot，无需独立 Web 服务器
