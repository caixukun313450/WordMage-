# WordMage 云服务器部署指南

**推荐免费方案**：Render（最简单）或 Oracle Cloud（长期免费 VPS）

---

## 方案 A：Render 免费托管（推荐，5 分钟上线）

无需自己买服务器，连 GitHub 即可部署。

### 特点

| 项目 | 说明 |
|------|------|
| 费用 | 免费 |
| 缺点 | 15 分钟无访问会休眠，首次打开需等 ~30 秒唤醒 |
| 适合 | 实训答辩、给同学演示 |

### 步骤

1. **把项目推到 GitHub**
   ```bash
   cd WordMage
   git init
   git add .
   git commit -m "WordMage deploy"
   git remote add origin https://github.com/你的用户名/WordMage.git
   git push -u origin main
   ```
   > 确保 `application-local.properties` 已在 `.gitignore` 中，不要上传 API Key。

2. **注册 Render**：https://render.com （可用 GitHub 登录）

3. **新建 Web Service**
   - 连接你的 GitHub 仓库
   - **Runtime** 选 **Docker**
   - **Plan** 选 **Free**
   - 添加环境变量：
     | Key | Value |
     |-----|-------|
     | `SPRING_PROFILES_ACTIVE` | `prod` |
     | `ZHIPU_API_KEY` | 你的智谱 API Key |

4. 点击 **Create Web Service**，等待构建（约 3–5 分钟）

5. 获得地址：`https://wordmage-xxxx.onrender.com`

## 方案 B：Oracle Cloud 永久免费 VPS

### 特点

| 项目 | 说明 |
|------|------|
| 费用 | 永久免费（4 OCPU ARM + 24GB 内存） |
| 缺点 | 注册需信用卡验证，国内注册有时较麻烦 |
| 适合 | 长期运行、答辩时不休眠 |

### 步骤

#### 1. 注册 Oracle Cloud

1. 打开 https://www.oracle.com/cloud/free/
2. 注册账号（需信用卡验证，不会扣费）
3. 创建 **Always Free** 的 ARM 虚拟机（Ubuntu 22.04）

#### 2. 配置安全组（重要）

在 Oracle 控制台 → 实例 → 子网 → 安全列表 → 添加入站规则：

| 来源 | 协议 | 端口 |
|------|------|------|
| 0.0.0.0/0 | TCP | 8080 |

#### 3. SSH 登录服务器

```bash
ssh ubuntu@你的公网IP
```

#### 4. 一键部署

**方式 1：Git 仓库部署**

```bash
curl -fsSL https://get.docker.com | sh
sudo usermod -aG docker $USER
# 重新登录 SSH 后继续

git clone https://github.com/你的用户名/WordMage.git /opt/wordmage
cd /opt/wordmage
cp .env.example .env
nano .env   # 填入 ZHIPU_API_KEY
docker compose up -d --build
```

**方式 2：本地上传**

在 Windows  PowerShell 中：

```powershell
# 打包项目（排除 target 和密钥）
cd D:\WordMage\WordMage
scp -r Dockerfile docker-compose.yml .env.example pom.xml mvnw mvnw.cmd .mvn src ubuntu@公网IP:/opt/wordmage/
```

然后在服务器上：

```bash
cd /opt/wordmage
cp .env.example .env && nano .env
docker compose up -d --build
```

#### 5. 访问

```
http://你的公网IP:8080
```

## 环境变量说明

| 变量 | 必填 | 说明 |
|------|------|------|
| `ZHIPU_API_KEY` | 是 | 智谱 AI API Key |
| `SPRING_PROFILES_ACTIVE` | 建议 `prod` | 启用生产配置 |
| `PORT` | 否 | 默认 8080，Render 会自动注入 |

---

## 常用运维命令

```bash
# 查看运行状态
docker compose ps

# 查看日志
docker compose logs -f

# 重启
docker compose restart

# 更新代码后重新部署
git pull
docker compose up -d --build

# 停止
docker compose down
```

---

## 安全提醒

1. **永远不要**把 API Key 提交到 GitHub
2. 生产环境只用环境变量或 `.env`（已在 `.gitignore`）
3. 答辩结束后可在云平台删除服务，避免 Key 被滥用
4. 建议重新生成已泄露过的 API Key

---

## 故障排查

| 问题 | 解决 |
|------|------|
| 外网打不开 | 检查云厂商安全组是否开放 8080 |
| 仍是演示模式 | 检查 `ZHIPU_API_KEY` 环境变量是否配置 |
| Render 构建失败 | 查看 Build Logs，确认 Dockerfile 路径正确 |
| Docker 内存不足 | Oracle 免费实例内存充足，一般无此问题 |
